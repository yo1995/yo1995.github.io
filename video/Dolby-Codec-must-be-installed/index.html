必须安装Dolby解码器

对于Adobe Premiere Pro CC 2017.4及以前版本

中文：必须安装Dolby解码器才能使用此功能。单击“确定”将安装并启用此编解码器，以供及时使用。

英文：”Dolby codec must be installed to use this feature. Click OK to install now.”

首先看两张图，这就是昨天一直困扰我的问题。在导入索尼相机拍摄的.MTS视频时发现导入的视频没有音轨。百度得到的结论是：无解……

不可能！遂一路按图索骥，找到了许多关于此问题的描述。原因主要可以归纳为几类：

  • 非正版授权及试用期的CC系列软件

  • Windows 7 系统或 macOS 10.11 及以前版本使用 Pr CC 2018

  • 多版本升级兼容

  • Media Encoder缓存异常

  • 其他由于非默认安装而导致的异常

对于低于2018的版本，解决方案非常简单：登录拥有正版授权的账号，重新打开软件即可。这种操作上的简洁再一次体现了正版软件的价值所在，或许也是大量优质软件与服务提供商逐渐转向云服务的原因之一。

解决方案

那么，我就是想试用，有办法吗？

首先，中文网络上长期存在的“Rovi TotalCode”的解法请勿轻易尝试。看看它的母公司MainConcept吧，想必总有人能回忆起当初标清时代被它支配的恐惧。安装它家Codec意味着你选择了闭源与管制，选择了稳定而非与时俱进的技术迭代。甚至，这种解法的“发明家”还强行把一个存在于CS世代的工具拿到五年后使用。拜托，HEVC都快成为主流标准啦！

其次，删除SLSCache以及类似的手工操作目录结构的方法请慎用。熟悉Adobe的朋友都知道这些文件夹是干什么用的。甚至很多长期试用版用户的hosts文件都是凌乱不堪的。在这样一个时代，想从一家逐步转型为云服务的软件公司搞到豁免，不如想办法把getty脱裤更靠谱了。

一个稳定且有效的解法：在2018年试用CC2015。重装系统，重新下载原始版试用软件，试用到天荒地老，绝对没问题。况且，连PS之父都无法完全用尽所有PS的功能，要啥自行车儿？

什么？你还想试用最新版？唉……这就触及我的知识盲区了。

办法当然是有的，挤一挤总还是有的。

  • 关键词：mc_dec_dd.dll \ dolbycodec.dll

具体可自行了解，但我并不认为有效。

  • 关键词：PP助手

思路类似，目前稳定情况未知。或许Adobe也会有闪退这样的机制？但实际上应该是打授权协议擦边球的办法。反正我是退出后重新0.9.2成功，目前无不良反应。

CC 2018.1及后续

请升级Windows 10 或安装旧版本。

旧版本可直接通过CC管理程序进行安装，点击右侧三角选择“其他版本”即可。安装旧版本后可以卸载新版本。

说实话，用了Windows 10 以后，感觉用不回Windows 7 了。事物总还是要发展的嘛。

总结

希望以后单位有Adobe订阅或者买得起订阅。

最后，侧目那些靠协议吃饭的厂家。AC-3让我想起了MD上ATRAC的结局。对，还有你，QuickTime。

ffmpeg大法好！
